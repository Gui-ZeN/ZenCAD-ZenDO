# Contribuindo

Obrigado pelo interesse no ecossistema Zen.

## Issues são muito bem-vindos

Bug, comportamento estranho, ideia de ferramenta, crítica de usabilidade: [abra
uma issue](https://github.com/Gui-ZeN/ZenCAD-ZenDO/issues). Relato de bug com
passos para reproduzir vale ouro, e é a forma de contribuição mais útil aqui.

## Pull requests de código não são aceitos

Não é falta de educação nem desinteresse — é uma decisão consciente de um
projeto mantido por uma pessoa só.

O ZenCAD e o Zendo são publicados sob GPL-3.0, e todo o código é de autoria de
um único autor. Isso mantém aberta a possibilidade de licenciar o projeto de
outras formas no futuro (o modelo do Qt: GPL para a comunidade, licença
comercial para quem precisar). Aceitar código de terceiros sem um acordo formal
de direitos fecharia essa porta de forma permanente — cada mudança de licença
passaria a exigir o consentimento de cada pessoa que já contribuiu.

Então: PRs de código são fechados com um agradecimento sincero. **Se a ideia for
boa, ela é implementada — e o crédito vai na issue.** É o mesmo caminho que o
SQLite adota, pelo mesmo motivo.

Se um dia isso mudar, este arquivo muda junto.
